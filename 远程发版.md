# 远程发版操作手册

> 目标：你在外地用任意电脑改代码 → push tag → 你爸的手机弹"发现新版本"提示，一键升级。

## 一次性配置（出门前在这台电脑做完）

### 1. 推到 GitHub

```bash
gh repo create XingShuHelper --private --source=. --remote=origin --push
```

注意：仓库可以是私有的，但 **Releases 上的 APK 需要公开访问**。GitHub 上私有仓库的 release assets 也是私有 URL，APK 内置直链下载会 404。两种解法：
- 简单：把仓库设为 Public（代码里没有 API key，安全）。
- 不想公开代码：单独建一个 public 仓库专门发 release，或自己挂 CDN/网盘。

本工程默认走"私有源码 + Public 仓库发版"——也可一开始直接全 public 省事。

### 2. 生成签名 keystore（如果还没生成）

```bash
keytool -genkeypair -v \
  -keystore release.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias xingshu \
  -storepass <你设的密码> -keypass <你设的密码> \
  -dname "CN=XingShu, O=Personal, C=CN"
```

**⚠️ 备份 release.jks 到至少 3 个地方**，丢了之后就再也无法发增量更新（用户必须卸载重装）：
- 1Password / Bitwarden（推荐，作为附件）
- iCloud Drive 或私有网盘
- 一台备用电脑或 U 盘

### 3. 配置 GitHub Actions Secrets

在仓库 Settings → Secrets and variables → Actions → New repository secret 里添加：

| Secret 名 | 内容 |
|---|---|
| `KEYSTORE_BASE64` | `base64 -i release.jks \| pbcopy` 后粘贴的内容 |
| `KEYSTORE_PASSWORD` | keystore 密码 |
| `KEY_ALIAS` | `xingshu`（即上面 -alias 的值）|
| `KEY_PASSWORD` | key 密码（一般和 storePassword 相同）|
| `DASHSCOPE_API_KEY` | 你的阿里云百炼 API key |

### 4. 让你爸先装一次最新版

第一次必须手动给他装一份带"自动检查更新"逻辑的 APK（之前的版本没有这套逻辑），方法不限：微信传文件、ADB、网盘都行。

之后每次更新就只需要 push tag，他打开 App 主页自动看到更新提示。

## 日常发版（出门后在任何电脑都行）

```bash
git commit -am "feat: 新功能"
git push origin main

# 升版本号 → push tag 触发 CI
git tag v1.0.1
git push origin v1.0.1
```

CI 会：
1. 解码 keystore；
2. 用 tag 版本号 build signed release APK；
3. 创建 GitHub Release 并把 APK 挂上去。

约 3-5 分钟后你爸打开 App 主页，会看到"发现新版本 v1.0.1"，点击下载并安装即可。

> 也可在 GitHub Actions 页面 → Release APK → Run workflow 手动触发，不一定要 push tag。

## 验证

- App 主页"行恕客服助手"标题下方有更新横幅区域。
- 如果当前已是最新版，横幅不显示（State.UpToDate / Idle 都隐藏）。
- 检查更新失败（网络问题、配置缺失）会以小字红色提示，不影响功能。

## 私有仓库 + 验证 token（进阶，可选）

如果坚持要保持仓库私有：在 `local.properties` 加 `GITHUB_TOKEN=<只读 PAT>`，
并在 `UpdateChecker.kt` 的 OkHttp 请求里把 `Authorization: token ...` 加上。
缺点：APK 反编译能拿到这个 token，token 应该用 fine-grained PAT 限定到这一个仓库的只读权限。
